System Concept Validation & Anti-Fraud Infrastructure

GridlockPulse operates completely via open-source public participation channels. To avoid system manipulation and prevent users from uploading incorrect bottleneck information to misguide other drivers, the upcoming MERN layer implements a two-step validation system.

First, reports are **GPS proximity-locked** using the browser's live geolocation tracking. This means a driver cannot report a false incident at Silk Board unless they are physically on that street segment. Second, the system features a **Time-To-Live (TTL) Decay algorithm**. All active marker coordinates automatically exhaust and delete themselves from the live map stream after a set timeframe unless continuously verified by other drivers hitting the 👍 validation counter.
